# extra-ge-qa-prompts
Reusable manual testing prompts for extra.ge project
## Included Prompts

### 1. Test Design Prompt
This prompt is used to generate detailed manual test cases for new features or user flows.

It helps cover:
- positive scenarios
- negative scenarios
- edge cases
- validations
- UI/UX checks
- business-critical flows

### 2. Exploratory Testing Prompt
This prompt is used to support exploratory testing sessions for pages, features, or user journeys.

It helps identify:
- hidden defects
- risky areas
- usability issues
- broken flows
- inconsistent behavior
- integration-related problems

### 3. Bug Analysis Prompt
This prompt is used to analyze reported bugs in a structured QA way.

It helps evaluate:
- possible root causes
- affected functionality
- user impact
- business impact
- severity and priority
- regression scope
- missing validation coverage

## How to Use

1. Open the relevant prompt file based on the QA task.
2. Copy the prompt content.
3. Replace the placeholder text with the actual feature description, page details, or bug information.
4. Paste the prompt into an AI tool.
5. Review the generated output and adjust it according to the project requirements.

## Project Context

These prompts are tailored for extra.ge, an e-commerce platform, and are intended for manual testing of common user flows such as:
- product listing
- product details page
- search and filters
- cart
- checkout
- login and registration
- favorites or wishlist
- pricing and discount behavior
- stock availability
- form validation
- error handling
- page navigation

## Benefits of This Library

This prompt library helps QA engineers:
- save time during test preparation
- create more structured and reusable QA outputs
- improve exploratory thinking
- analyze bugs more deeply
- maintain better testing consistency across features
